# Sentinel Briefing
*2026-03-02T03:00:00.000Z · provider: claude-sonnet-4-6*

## What Changed

**Todoist triage flow — complete.** Three commits delivered the full loop: inline priority badges on triage cards, dismiss now calls Todoist to close the task (not just hide it in Clarity), and approve writes the chosen priority back to Todoist via REST v2. Triage scoring tests were added and the Todoist score threshold was removed so all items surface.

**Gmail dismiss → trash.** Dismissed emails are now trashed in Gmail rather than archived. This prevents the incremental sync from re-fetching them and re-queuing them in triage — a recurring re-sync bug that had been chasing the team.

**Mobile UX sprint.** Six files touched across one commit: `touch-action: manipulation` on all buttons/labels eliminates the 300ms iOS tap delay globally; collapsible sections got `min-h-[44px]` touch targets; the coach submit button grew from 28px to 36px; triage card action rows stack vertically on mobile with taller priority pills; the approve modal is now constrained to `90dvh` so the "Create Task" button stays visible on short screens; and the chat sessions sidebar was wrapped in a `Sheet` component, accessible via a `PanelLeft` trigger, fixing the previously inaccessible sidebar on mobile.

## Likely Next

With triage and mobile polish complete, the most natural next area is **Plaid / spending improvements**. Plaid was recently pulled from cron sync (manual-only), which may frustrate if transactions are stale. Restoring automated Plaid sync — or at minimum a UI indicator of last-sync time — is a likely next step. Alternatively, the developer may begin the **Apple companion bridge** (still listed as "planned" in CLAUDE.md), or address **push notifications** which were deferred to v1.1.

## Watch Out

- **Deprecated Sync API still in use.** `src/lib/integrations/todoist.ts` still imports `TODOIST_SYNC = "https://api.todoist.com/sync/v9"` alongside the REST v2 usage. This was flagged in the previous sentinel. The Sync API is needed for certain bulk operations but it's worth auditing whether any REST v2 equivalents can replace remaining Sync API calls.
- **Plaid cron removed.** Transactions will go stale unless the user manually triggers sync. If spending views become a daily-use feature, silent staleness could confuse.
- **Mobile Sheet sidebar state.** The new chat Sheet on mobile is uncontrolled — navigating to a chat session from the sheet may not auto-close it. Worth testing the open/close flow on a real device.
